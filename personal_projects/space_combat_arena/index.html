About

Github project link

My main motivation for this project was that I wanted to try to make some game using the 3d rendering and physics engine that I had wrote (which I have a repo for here).

Setting the game in space was a workaround to the limitations of the physics engine I had written. I had no sleep system and an slow collision detection algorithm, so the number of dynamic objects that could exist at once with reasonable performance was limited. By setting the game in space, the vast majority of the time almost all of the physics objects are not close enough to collide which other. On most ticks all potential collison pairs are discarded by the octtree I used for broadphase, rather than requiring narrow phase collision detection.

The final version of the game is a infinite-wave game, where the only objective is to see how far you can make it against infinite waves of increasingly larger and more difficult enemy spaceships. The original vision when I was building this was very different though- it was going to be a type of idle game where the player would mine the asteroids, which on destruction spawned ores of various different value. The player then would bring the ores back to a space station which they would sell them, and use the money to buy various upgrades to their mining, weapons, speed, etc. To keep things interesting, the player would be periodically attacked by enemy ships and need to defend themselves.

I made it pretty far with this vision; the space station is fully modeled and had working turrets that would track and shoot at enemies. Mining asteroids and collecting the ores worked, and the shop menu was functioning (although ugly). To ensure there were plenty of asteroids for the player to mine, I wrote a chunking system that used a hash of the coords to seed a random number generator when calculating the position and number of asteroids created. This meant the same asteroids would created when loading, unloading, and reloading a chunk. That system is still in the game, but it's completely unnecessary. A random tidbit related to it is that I was taking a statistics class the fall of 2020, and had just learned about poisson distributions. I was able to use that to get a more "realistic" randomness for the number of asteroids per chunk.

The issue I ended up running into is that the game is that the game loop of mining asteroids did not seem like it would be compelling. But I had spent a bunch of time testing fighting against the enemies however, and I knew that was fun. I decided to cut everything mining / economy related, and just make the combat the entire game loop, which resulted in the final game.
