About

Github project link

I mostly wrote this as I thought it would be interesting to write a software floating point implementation. Using those with a mandelbrot is a pretty natural fit, with normal floating point numbers you can only zoom so far in before you reach the precision limit. With a software floating point, you can increase the level of precision as needed as you go deeper.

One small detail I was proud of was that I independently came up with the idea that for division, I could truncate the current accuracy to a double-precision float and use the hardware to compute an initial estimate, and then refine it using Newtons method to the needed level of precision. I checked online after and found it was an already known optimization, but I was happy that I had thought of it myself.

The other trick I use is pertubations (described here), which is basically calculating a few anchor points in full precision, and estimating the values of nearby points using an estimate based on the original points orbit. I have a pretty naive implementation of this though, so it sometimes results in artifacts.

Note the interractive demo is a little bit slow as there is no multithreading. I was mostly interested in optimizing for non-realtime renders, so only bothered to parallelize on the multi-frame level.
