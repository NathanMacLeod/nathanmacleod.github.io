About

Github project link

This project has been my attempt to build a more robust physics engine compared to the previous one I used in the Space Combat Arena project. I enjoy working on game physics as they involve a lot of mathematics and low level multi-threaded programming that I personally find very interesting.

The engine supports convex hulls, spheres, cylinders, capsules for collision primitives. For static or kinematic objects triangle meshes are also supported. Collision detection between the convex objects uses SAT with a Gauss Map optimization to avoid n^2 edge vs edge checks, using the method described in this presentation by Dirk Gregorious. Contacts are persisted accross frames to allow for warm starting. The contact is identified using an id system based on the features that generate the contact, e.g. edge 1 of object A + face 3 of object B.

For broad phase, the engine uses an AABB tree as described in this presentation by Erin Catto

The solver is a hybrid-solver using sequential impulses supporting substeps. A block solver computes a direct solution for groups of holonomic constraints via LDL decomposition as described in this presentation by Maciej Mizerski. Positional error is solved via dual impulses; psuedo impulses are solved for alongside the normal impulses and applied for a single frame. Multiple constraint types are supported by the solver: contact, friction, distance, ball socket, cone, twist, hinge, sliding hinge, sliding, motor, piston, & weld.

Gyroscopic motion is simulated by solving for internal gyroscopic forces implicitly while integrating velocities, using the method described in this presentation by Erin Catto.

The engine is multi-threaded, using a thread pool and custom task scheduling system.
